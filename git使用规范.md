# 0 git使用规范
本文档用来规范版本控制工具git在本项目中的使用规范。
# 1 分支的使用
## 1.1 普通文件的管理
**普通文件**，例如`.gitignore`等。
每次更改，需要打一个分支，更改完成后，提交，合并到`master`分支中。
分支命名示例：`file-NO`
分支命名列表见附录。
## 1.2 项目文件的管理
**项目文件**，独立于练习之外的文件。
每次更改，需要打一个分支，更改完成后，提交，合并到`master`分支中。
分支命名示例：`l_log-1`
分支命名列表见附录。
## 1.3 章节练习的管理
**章节练习**，普通的单元练习。
每次更改，需要打一个分支，更改完成后，提交，合并到`master`分支中。
分支命名示例：`chapterN-1`
分支命名列表见附录。
